📡 通訊協定 (Protocol)

網路通訊的規則與標準 — 資料如何在網路中正確傳輸

什麼是通訊協定？

通訊協定（Protocol）是網路中設備之間通訊時必須遵守的一組規則和約定。就像人類交談需要共同的語言，電腦之間通訊也需要共同的協定來確保資料能正確傳送和理解。

關鍵概念：通訊協定義定了資料的格式、傳送順序、錯誤處理機制、以及資料傳輸的速率控制等規範。

通訊協定的三要素

  • 語法 (Syntax)：資料的格式與結構，包括欄位長度、編碼方式等
  • 語意 (Semantics)：每個欄位的意義與控制資訊的解釋方式
  • 時序 (Timing)：資料傳送的時機、速度匹配與順序控制

OSI 七層參考模型

OSI（Open Systems Interconnection）模型由ISO（國際標準組織）制定，將網路通訊分為七個層次，每層負責不同的功能：

層級 名稱 主要功能 常見協定/技術
Layer 7 應用層 (Application) 提供使用者應用程式的網路服務 HTTP, FTP, SMTP, DNS
Layer 6 表達層 (Presentation) 資料格式轉換、加密解密、壓縮 SSL/TLS, JPEG, MPEG
Layer 5 會議層 (Session) 建立、管理、終止會話連線 NetBIOS, RPC
Layer 4 傳輸層 (Transport) 端對端傳輸、流量控制、錯誤恢復 TCP, UDP
Layer 3 網路層 (Network) 路由選擇、封包轉送、邏輯定址 IP, ICMP, ARP
Layer 2 資料連結層 (Data Link) 幀傳輸、實體定址(MAC)、錯誤偵測 Ethernet, PPP, HDLC
Layer 1 實體層 (Physical) 位元傳輸、電氣/光學訊號規範 RS-232, RJ-45, 光纖

記憶口訣：由上到下 — 應用、表達、會議、傳輸、網路、資料連結、實體（應表會傳網資實）

TCP/IP 四層模型

TCP/IP 模型是實際網際網路使用的架構，將OSI七層簡化為四層：

TCP/IP 層 對應 OSI 層 主要協定
應用層 應用層 + 表達層 + 會議層 HTTP, FTP, SMTP, DNS, SSH
傳輸層 傳輸層 TCP (可靠), UDP (不可靠)
網際網路層 網路層 IP, ICMP, ARP
網路存取層 資料連結層 + 實體層 Ethernet, Wi-Fi, PPP

TCP vs UDP 比較

TCP (傳輸控制協定)
  • 面向連線 (Connection-oriented)
  • 可靠傳輸 (三次握手建立連線)
  • 流量控制 (滑動視窗)
  • 擁塞控制 (慢啟動、擁塞避免)
  • 資料按序到達
  • 適用：HTTP, FTP, Email
UDP (使用者資料報協定)
  • 無連線 (Connectionless)
  • 不可靠傳輸 (盡最大努力)
  • 無流量控制
  • 無擁塞控制
  • 低延遲、高效率
  • 適用：DNS, VoIP, 視訊串流

流量控制與錯誤控制

流量控制 (Flow Control)

確保傳送方的速率不會超過接收方的處理能力：

  • Stop-and-Wait：傳送一個封包後等待確認，再傳送下一個
  • Sliding Window（滑動視窗）：允許連續傳送多個封包，視窗大小決定同時可傳送的數量
  • Go-Back-N：出錯時從錯誤的封包開始重新傳送
  • Selective Repeat：只重新傳送出錯的封包

錯誤偵測 (Error Detection)

  • Parity Check（同位元檢查）：加入一個同位元使1的個數為奇數或偶數
  • Checksum（總和檢查碼）：將資料分段相加後取補碼
  • CRC（循環冗餘檢查）：使用多項式除法產生餘數作為檢查碼，偵測能力最強

重要：CRC 是最廣泛使用的錯誤偵測技術，乙太網路幀的FCS欄位就使用CRC-32演算法。

封包交換 vs 電路交換

特性 電路交換 (Circuit Switching) 封包交換 (Packet Switching)
連線建立 需要預先建立專用路徑 不需要預先建立路徑
頻寬使用 固定分配，可能浪費 動態共享，效率高
延遲 傳輸延遲低且穩定 可能有排隊延遲
典型應用 傳統電話網路 網際網路

重要網路協定摘要

協定 全名 功能 埠號
HTTP Hypertext Transfer Protocol 網頁傳輸 80
HTTPS HTTP Secure 加密網頁傳輸 443
FTP File Transfer Protocol 檔案傳輸 21
SMTP Simple Mail Transfer Protocol 電子郵件傳送 25
DNS Domain Name System 域名解析 53
DHCP Dynamic Host Configuration Protocol 自動分配IP位址 67/68
ARP Address Resolution Protocol IP位址轉MAC位址 N/A
ICMP Internet Control Message Protocol 網路錯誤報告與偵測 N/A

🔌 乙太網路 (Ethernet)

最廣泛使用的區域網路技術 — 從10Mbps到100Gbps的演進

乙太網路概述

乙太網路（Ethernet）是由Xerox PARC在1970年代開發的區域網路技術，現已成為IEEE 802.3標準。它是目前最普遍使用的LAN技術，從辦公室到資料中心都廣泛採用。

核心概念：乙太網路定義了實體層（Layer 1）和資料連結層（Layer 2）的媒體存取控制（MAC）子層的規範。

乙太網路的演進

1976年 — 原始乙太網路

Robert Metcalfe提出，2.94 Mbps，使用同軸纜線

1983年 — 10BASE5 (Thick Ethernet)

10 Mbps，粗同軸纜線，最大長度500公尺

1985年 — 10BASE2 (Thin Ethernet)

10 Mbps，細同軸纜線，最大長度185公尺

1990年 — 10BASE-T

10 Mbps，雙絞線（UTP Cat3），星狀拓扑，最大100公尺

1995年 — 100BASE-TX (Fast Ethernet)

100 Mbps，UTP Cat5，保持相同幀格式

1999年 — 1000BASE-T (Gigabit Ethernet)

1 Gbps，UTP Cat5e，四對線全雙工傳輸

2002年 — 10 Gigabit Ethernet

10 Gbps，主要使用光纖，後來也有銅纜版本

2010年以後 — 40G/100G/400G Ethernet

高速資料中心應用，使用多條通道聚合

乙太網路命名規則

乙太網路標準的命名格式為：速度 BASE 傳輸媒介

標準名稱 速度 傳輸媒介 最大距離
10BASE5 10 Mbps 粗同軸纜線 500m
10BASE2 10 Mbps 細同軸纜線 185m
10BASE-T 10 Mbps UTP (Cat3) 100m
100BASE-TX 100 Mbps UTP (Cat5) 100m
100BASE-FX 100 Mbps 多模光纖 2km
1000BASE-T 1 Gbps UTP (Cat5e) 100m
1000BASE-SX 1 Gbps 多模光纖 550m
1000BASE-LX 1 Gbps 單模光纖 5km

CSMA/CD 存取控制機制

傳統乙太網路（半雙工模式）使用 CSMA/CD（Carrier Sense Multiple Access with Collision Detection）作為媒體存取控制方法：

CSMA/CD 運作步驟：

  1. 偵聽 (Carrier Sense)：傳送前先偵聽線路是否空闲
  2. 傳送：若線路空闲則開始傳送資料
  3. 碰撞偵測 (Collision Detection)：傳送時持續偵測是否發生碰撞
  4. 碰撞處理：若偵測到碰撞，立即停止傳送並發出Jam訊號
  5. 隨機等待：使用Binary Exponential Backoff演算法等待隨機時間後重傳

注意：全雙工乙太網路（使用交換器）不需要CSMA/CD，因為發送和接收使用不同的線對，不會發生碰撞。

乙太網路幀格式 (Ethernet Frame)

標準乙太網路幀（IEEE 802.3）的結構如下：

| Preamble | SFD | Destination MAC | Source MAC | Type/Length | Data (Payload) | FCS | | 8 bytes | 1B | 6 bytes | 6 bytes | 2 bytes | 46-1500 bytes | 4B |
欄位 大小 說明
Preamble（前導碼） 7 bytes 同步訊號，10101010交替模式
SFD（幀起始分隔符） 1 byte 10101011，標示幀的開始
Destination MAC 6 bytes 目標设备的MAC位址
Source MAC 6 bytes 來源设备的MAC位址
Type/Length 2 bytes 上層協定類型 或 資料長度
Data（資料） 46-1500 bytes 有效負載，最小46 bytes（不足需填充）
FCS（幀檢查序列） 4 bytes CRC-32錯誤偵測碼

最小幀大小：64 bytes（不含Preamble）— 確保CSMA/CD能正確偵測碰撞。
最大幀大小：1518 bytes（標準），稱為MTU（Maximum Transmission Unit）。

MAC 位址

MAC（Media Access Control）位址是全球唯一的48位元硬體位址，燒錄在網路卡中：

  • 格式：12位十六進制數字，如 00:1A:2B:3C:4D:5E
  • 前24位元：OUI（Organizationally Unique Identifier），由IEEE分配給製造商
  • 後24位元：製造商分配的介面特定識別碼
  • Broadcast位址：FF:FF:FF:FF:FF:FF，傳送到所有設備
  • Multicast位址：第一個位元的最低有效位為1

全雙工 vs 半雙工

全雙工 (Full-Duplex)
  • 可同時發送和接收資料
  • 使用交換器 (Switch)
  • 不會發生碰撞
  • 不需要 CSMA/CD
  • 頻寬利用率：100%
  • 例：1 Gbps 全雙工 = 2 Gbps 總吞吐量
半雙工 (Half-Duplex)
  • 同一時間只能發送或接收
  • 使用集線器 (Hub)
  • 可能發生碰撞
  • 需要 CSMA/CD
  • 頻寬利用率：< 100%
  • 例：100 Mbps 半雙工 = 100 Mbps 共享

乙太網路物理拓扑

  • 匯流排拓扑 (Bus)：使用同軸纜線，所有設備連接在同一條主幹線上（早期10BASE5/10BASE2）
  • 星狀拓扑 (Star)：所有設備連接到中央設備（Hub或Switch），現代最常用（10BASE-T及以上）
  • 延伸星狀 (Extended Star)：多個星狀網路串接而成，用於大型網路

🏗️ TIA 標準化佈線

TIA/EIA-568 商業建築電信佈線標準 — 結構化佈線的基石

TIA/EIA-568 標準概述

TIA/EIA-568 是由電信工業協會（TIA）和電子工業協會（EIA）共同制定的商業建築電信佈線標準。此標準定義了結構化佈線系統的設計、安裝和測試規範。

標準演進：TIA/EIA-568-A (1995) → TIA/EIA-568-B (2001) → TIA-568-C (2009) → TIA-568-D (2015)

標準的主要目標

  • 建立通用的電信佈線系統，支持多廠商產品互通
  • 定義佈線系統的最低技術要求
  • 提供規劃和安裝的指導方針
  • 確保佈線系統能支持當前和未來的應用

結構化佈線六大子系統

TIA-568將佈線系統分為六個子系統：

1. 進線間 (Entrance Facility)

建築物外部線路進入的接入點，包含避雷和保護裝置。

  • 連接外部電信服務
  • 包含過電壓保護
  • 通常與設備間相連

2. 設備間 (Equipment Room)

放置電信設備（交換器、路由器、伺服器）的集中空間。

  • 主配線架 (MDF) 所在位置
  • 環境控制（溫度、濕度）
  • 電力與備援系統

3. 骨幹佈線 (Backbone Cabling)

連接各樓層電信間的垂直佈線系統。

  • 使用光纖或大對數UTP
  • 連接MDF到IDF
  • 需考慮防火等级

4. 電信間 (Telecommunications Room)

每層樓的配線間，放置该楼层的配線架和網路設備。

  • 中間配線架 (IDF) 所在位置
  • 水平佈線的集中點
  • 需足夠空間維護

5. 水平佈線 (Horizontal Cabling)

從電信間到工作區出口的水平方向佈線。

  • 最大長度90公尺（固定佈線）
  • 加上跳接線總長不超過100公尺
  • 常用Cat5e/Cat6 UTP

6. 工作區 (Work Area)

從資訊插座到终端设备的區域。

  • 包含資訊插座 (Outlet)
  • 設備跳接線 (Patch Cord)
  • 最大跳接線長度5公尺

商業環境最常用

直通線 (Straight-through)：兩端使用相同標準（A-A或B-B），用於連接不同類型設備（如PC到Switch）。

交叉線 (Crossover)：一端A一端B，用於連接相同類型設備（如PC到PC、Switch到Switch）。*注意：現代設備大多支援Auto-MDIX，自動偵測並調整。

總通道長度 100 公尺 端到端最大允許長度

纜線測試參數

安裝完成後需要對纜線進行測試認證，主要測試參數包括：

  • 接線圖 (Wire Map)：確認每根線的連接正確性，無開路、短路、錯對、串對
  • 長度 (Length)：確認不超過100公尺限制
  • 衰減 (Attenuation)：訊號在傳輸過程中的減弱程度
  • 近端串擾 (NEXT)：同一端發送線對對接收線對的干擾
  • 遠端串擾 (FEXT)：發送端對遠端接收端的串擾
  • 回波損耗 (Return Loss)：因阻抗不匹配造成的訊號反射
  • 延遲偏移 (Delay Skew)：四對線之間傳輸延遲的差異

🌐 區域網路 (LAN)

LAN設計、設備、VLAN與無線網路的完整知識體系

區域網路 (LAN) 定義與特性

區域網路（Local Area Network, LAN）是在有限地理範圍內（如一棟建築、一個校園）連接多台電腦和設備的通訊網路。

LAN 的主要特性

  • 地理範圍小：通常在幾公尺到幾公里內
  • 高傳輸速率：從10 Mbps到100 Gbps
  • 低錯誤率：傳輸品質高，錯誤率低
  • 私有管理：通常由單一組織管理和擁有
  • 标准技術：主要使用IEEE 802標準系列

LAN vs WAN 比較

特性 LAN WAN
範圍 有限區域（建築/校園） 廣域（城市/國家/全球）
速率 高（10 Mbps - 100 Gbps） 較低（受電信頻寬限制）
擁有者 單一組織 多家電信業者
錯誤率 低 較高
拓樸 星狀、匯流排、環狀 網狀、樹狀

網路拓樸結構

⭐ 星狀拓樸 (Star)

  • 所有節點連接中央設備
  • 單一節點故障不影響其他
  • 中央設備故障則全網中斷
  • 易於管理和擴充
  • 最常見的LAN拓樸

🔗 匯流排拓樸 (Bus)

  • 所有節點共享一條主幹線
  • 終端需要終端電阻
  • 主幹斷裂則全網中斷
  • 碰撞問題較嚴重
  • 早期Ethernet使用

⭕ 環狀拓樸 (Ring)

  • 節點首尾相連成環
  • 資料單方向傳輸
  • 使用Token傳遞控制
  • 單一節點故障影響全網
  • Token Ring網路使用

🌳 樹狀拓樸 (Tree)

  • 星狀拓樸的延伸
  • 階層式結構
  • 易於擴充
  • 適合大型網路
  • 根節點故障影響大

🕸️ 網狀拓樸 (Mesh)

  • 每個節點互連
  • 高冗餘性與可靠性
  • 成本最高
  • 多路徑路由選擇
  • WAN常用

📡 混合拓樸 (Hybrid)

  • 多種拓樸組合
  • 靈活適應需求
  • 實務上最常用
  • 可優化成本和效能
  • 企業網路常見

IEEE 802 標準系列

IEEE 802是定義LAN和MAN技術的標準家族：

標準 名稱 描述
802.1 高層介面 網路管理、VLAN、橋接
802.2 LLC 邏輯連結控制子層
802.3 Ethernet CSMA/CD乙太網路
802.5 Token Ring 令牌環網路（已淘汰）
802.11 Wi-Fi 無線區域網路
802.15 Bluetooth 無線個人區域網路
802.16 WiMAX 寬頻無線存取

橋接器 (Bridge) 與交換器 (Switch)

橋接器

橋接器工作在資料連結層（Layer 2），用於連接兩個或多個網路區段：

  • 根據MAC位址轉送幀
  • 建立MAC位址學習表
  • 分割碰撞域（Collision Domain）
  • 不分割廣播域（Broadcast Domain）
  • 通常只有2-4個埠

交換器 (Switch)

交換器是多埠橋接器，現代LAN的核心設備：

  • 每個埠是獨立的碰撞域
  • 支援全雙工通訊
  • 硬體ASIC高速轉送
  • 支援VLAN功能
  • 通常有8-48個或更多埠
  • 支援生成樹協定（STP）防止迴路

重要區別：
• Hub（集線器）— Layer 1，廣播到所有埠，共享頻寬
• Switch（交換器）— Layer 2，根據MAC位址轉送，獨立頻寬
• Router（路由器）— Layer 3，根據IP位址轉送，分割廣播域

VLAN 虛擬區域網路

VLAN（Virtual LAN）允許在單一實體交換器上建立多個邏輯網路：

VLAN 的優點

  • 安全性：不同VLAN之間無法直接通訊
  • 效能：減少廣播風暴的影響範圍
  • 管理彈性：邏輯分組不受實體位置限制
  • 成本節省：不需要額外硬體即可分割網路

VLAN 分類方式

  • Port-based VLAN：根據交換器埠號分配VLAN
  • MAC-based VLAN：根據設備MAC位址分配
  • Protocol-based VLAN：根據網路協定類型分配
  • Subnet-based VLAN：根據IP子網路分配

IEEE 802.1Q 標籤

802.1Q是VLAN標籤標準，在乙太網路幀中插入4 byte的VLAN標籤：

| Destination | Source | 802.1Q Tag | Type | Data | FCS | | 6 bytes | 6B | 4 bytes | 2B | ... | 4B | 802.1Q Tag 結構： | TPID (0x8100) | Priority | CFI | VLAN ID (12 bits) | | 2 bytes | 3 bits | 1bit| 0-4094 |

注意：VLAN ID範圍是1-4094，VLAN 0和4095保留。加入VLAN標籤後，幀的最大大小從1518 bytes增加到1522 bytes。

生成樹協定 (STP)

STP（Spanning Tree Protocol, IEEE 802.1D）用於防止交換器網路中的迴路問題：

  • 目的：在有冗餘路徑的網路中防止廣播風暴和幀重複
  • 運作原理：建立一棵無迴路的樹狀拓樸，阻斷冗餘路徑
  • 根橋 (Root Bridge)：選舉Bridge ID最小的交換器作為根橋
  • 埠角色：
    • Root Port：連接根橋的最佳路徑埠
    • Designated Port：負責轉送資料到該區段的埠
    • Blocked Port：被阻斷以防止迴路的埠

進階版本：
• RSTP (802.1w) — 快速生成樹，收斂時間更短
• MSTP (802.1s) — 多實例生成樹，支援多個STP實例

無線區域網路 (WLAN)

無線區域網路（Wireless LAN, WLAN）使用無線電波代替纜線連接設備，基於IEEE 802.11標準。

802.11 標準演進

標準 頻段 最大速率 頻寬 特色
802.11b 2.4 GHz 11 Mbps 20 MHz 最早普及的Wi-Fi
802.11g 2.4 GHz 54 Mbps 20 MHz OFDM調變
802.11a 5 GHz 54 Mbps 20 MHz 較少干擾
802.11n
(Wi-Fi 4)		 2.4/5 GHz 600 Mbps 20/40 MHz MIMO技術
802.11ac
(Wi-Fi 5)		 5 GHz 6.9 Gbps 20-160 MHz Wave2 MU-MIMO
802.11ax
(Wi-Fi 6)		 2.4/5 GHz 9.6 Gbps 20-160 MHz OFDMA、高效率
802.11be

WLAN 操作模式

基礎架構模式 (Infrastructure)

  • 設備通過存取點（AP）連接
  • AP管理無線傳輸
  • 可連接有線網路
  • 最常見的使用方式
  • BSS（基本服務組）

Ad-hoc 模式 (IBSS)

  • 設備之間直接連接
  • 不需要存取點
  • 臨時性網路
  • 適合少量設備
  • 獨立基本服務組

碰撞域 vs 廣播域

設備 碰撞域 廣播域
Hub（集線器） 所有埠共享1個 所有埠共享1個
Bridge（橋接器） 每個埠獨立 所有埠共享1個
Switch（交換器） 每個埠獨立 所有埠共享1個（無VLAN時）
Router（路由器） 每個埠獨立 每個埠獨立

關鍵規則：每增加一個路由器介面，就增加一個廣播域。交換器每個埠是一個獨立的碰撞域。VLAN可以將一個交換器的廣播域分割為多個。

⚡ 第五章：Ethernet 進階

乙太網路進階技術 — 交換器內部運作、連結聚合、PoE 與 Jumbo Frame

交換器轉送模式

交換器根據接收幀的方式不同，有以下三種轉送模式：

模式 運作方式 延遲 錯誤檢查
Store-and-Forward
儲存後轉送		 完整接收整個幀後檢查FCS，無誤才轉送 高（完整接收延遲） 完整CRC檢查
Cut-Through
直接轉送		 讀取目的MAC位址後立即轉送，不等待完整接收 低 無錯誤檢查
Fragment-Free
無碎片轉送		 接收前64 bytes後再轉送，過濾碰撞碎片 中 檢查前64 bytes

實務建議：現代交換器大多預設使用 Store-and-Forward 模式以確保資料完整性。部分高階交換器支援動態切換模式，根據網路錯誤率自動調整。

MAC 位址表學習機制

交換器透過學習來源MAC位址建立轉送資料庫（Forwarding Database, FDB）：

  1. 學習 (Learning)：接收幀時，記錄來源MAC位址和對應的埠號
  2. 轉送 (Forwarding)：查詢目的MAC位址，若找到則只轉送到對應埠
  3. 泛洪 (Flooding)：若目的MAC位址不在表中，則轉送到所有埠（除來源埠外）
  4. 老化 (Aging)：MAC位址表項目有TTL（通常300秒），逾時則刪除
  5. 過濾 (Filtering)：若來源和目的MAC在同一埠，則不轉送

MAC位址表攻擊：攻擊者可偽造大量MAC位址填滿交換器MAC表，導致交換器進入「失效開放」（Fail-Open）模式，被迫泛洪所有幀，讓攻擊者得以嗅探網路流量。

鏈路聚合 (Link Aggregation)

Link Aggregation（IEEE 802.3ad / 802.1AX）將多條實體鏈路結合成一條邏輯鏈路：

優點
  • 頻寬疊加（4條1GbE = 4 Gbps）
  • 鏈路冗餘（一條斷線不中斷）
  • 负载均衡（流量分散到多條鏈路）
  • 成本效益高於升級更高頻寬
實作方式
  • 靜態 (Static)：手動設定兩端交換器
  • LACP (動態)：使用Link Aggregation Control Protocol自動協商
  • 最多支援8條主動鏈路
  • 支援新增/移除而不中斷服務

Hash演算法：鏈路聚合使用Hash演算法（根據MAC/IP/Port）決定流量分配到哪條實體鏈路，確保同一連線的封包走同一條路徑。

巨型幀 (Jumbo Frame)

Jumbo Frame 是超過標準乙太網路MTU（1500 bytes）的大型幀，通常支援 9000 bytes 的承載量：

特性 標準幀 Jumbo Frame
最大Payload 1500 bytes 9000 bytes
總幀大小 1518 bytes ~9022 bytes
傳輸效率 較低（更多幀頭開銷） 較高（減少幀頭比例）
CPU負載 較高（需處理更多中斷） 較低（中斷次數減少）
適用場景 通用網路 資料中心、iSCSI、NAS

注意：Jumbo Frame 需要路徑上所有設備（交換器、路由器、網卡）都支援且啟用，否則會造成封包被丟棄。

乙太網路供電 (PoE)

Power over Ethernet (PoE, IEEE 802.3af/at/bt) 透過乙太網路纜線同時傳輸資料和電力：

標準 最大功率 推出年份 典型應用
802.3af (PoE) 15.4 W 2003 IP電話、無線AP
802.3at (PoE+) 30 W 2009 PTZ攝影機、視訊電話
802.3bt (PoE++) 60-100 W 2018 LED照明、資訊站、筆電

PoE好處：減少電源佈線成本、集中電力管理（UPS備援）、設備部署位置更靈活。

交換器堆疊 (Stacking)

交換器堆疊技術將多台實體交換器透過堆疊電纜連接，組成單一邏輯交換器：

  • 堆疊頻寬：通常 40-160 Gbps 專屬堆疊介面
  • 單一管理：透過單一IP管理所有成員交換器
  • 高可用性：主交換器故障時，備份交換器自動接管
  • 簡化拓撲：減少STP實例和路由協定鄰居數量
  • 跨堆疊鏈路聚合：可將鏈路聚合到不同交換器成員

堆疊 vs 獨立：堆疊適用於存取層和匯聚層，提供高密度埠數和簡化管理。核心層通常使用獨立的模組化交換器以獲得最佳效能。

🏙️ 第七章：LAN 進階

區域網路進階設計 — 企業級架構、冗餘、QoS、網路管理與SDN

企業級LAN設計架構

現代企業LAN採用三層式階層設計模型：

層級 角色 設備 特性
核心層 (Core) 高速骨幹轉送 高階模組化交換器 高吞吐量、低延遲、高可靠性
匯聚層 (Distribution) 政策實施、路由聚合 L3交換器 VLAN路由、安全過濾、QoS
存取層 (Access) 終端設備連接 L2交換器 PoE、VLAN存取、埠安全

Two-Tier 設計（精簡模型）

較小型的網路可以使用兩層設計：

  • Collapsed Core：核心和匯聚合併為一層
  • 存取層：連接到合併的核心/匯聚層交換器
  • 適用於中小型企業或單一建築

鏈路冗餘與故障轉移

網路高可用性需要多層次的冗餘設計：

設備冗餘

  • 雙核心交換器（Active/Active 或 Active/Standby）
  • 雙電源供應器（1+1 冗餘）
  • 雙上鏈（Dual Homing）到不同匯聚交換器

鏈路冗餘協定

  • STP (802.1D)：標準生成樹，收斂時間 30-50 秒
  • RSTP (802.1w)：快速生成樹，收斂時間 1-3 秒
  • MSTP (802.1s)：多實例生成樹，可负载均衡
  • REP (Resilient Ethernet Protocol)：思科私有，快速收斂

設計原則：網路設計應遵循「無單點故障」原則。關鍵路徑的每個環節（設備、鏈路、電源）都要有備援。

服務品質 (QoS)

QoS（Quality of Service）管理網路頻寬，確保重要流量獲得優先處理：

QoS模型 描述 優點 缺點
Best Effort 盡力而為，無優先級 簡單 無品質保證
IntServ
整合服務		 預先保留頻寬（RSVP） 精確保證 擴展性差
DiffServ
區分服務		 封包標記優先級（DSCP） 擴展性好 逐跳行為可能不同

QoS 佇列機制

  • FIFO：先進先出，無優先級
  • PQ (Priority Queuing)：絕對優先，高優先級可能阻塞低優先級
  • WFQ (Weighted Fair Queuing)：加權公平佇列，保證最小頻寬
  • CBWFQ (Class-Based WFQ)：依流量類別分配頻寬
  • LLQ (Low Latency Queuing)：結合PQ和CBWFQ，確保即時流量延遲

SNMP 網路管理

SNMP（Simple Network Management Protocol）是網路設備管理的標準協定：

SNMP 元件
  • NMS：網路管理系統（Server）
  • Agent：設備端的管理軟體
  • MIB：管理資訊庫（資料結構）
  • OID：物件識別碼（樹狀結構）
SNMP 版本
  • v1：基本功能，明文社群字串
  • v2c：增加批量檢索
  • v3：加密和認證安全性

SNMP 操作

  • Get：NMS讀取Agent的變數值
  • Set：NMS設定Agent的變數值
  • Trap：Agent主動通知NMS事件（如介面down）
  • Walk：遍歷MIB樹狀結構擷取所有資訊

網路監控與分析工具

工具 協定/技術 用途
NetFlow Cisco專屬 流量記錄分析（IP、埠、協定）
sFlow 取樣式 即時流量監控與統計
IPFIX IETF標準 NetFlow的標準化版本
RMON 遠端監控 網路區段流量統計與告警
Wireshark 封包擷取 深度封包分析與除錯
PRTG / Zabbix SNMP / 代理 綜合網路監控平台

SDN 軟體定義網路

SDN（Software-Defined Networking）將網路控制層與資料轉發層分離：

傳統網路
  • 控制和轉發在同一設備
  • 分散式控制平面
  • 每台設備獨立決策
  • 協定收斂時間較長
  • 管理和除錯困難
SDN 網路
  • 控制平面集中化（Controller）
  • 交換器僅負責轉發
  • OpenFlow 南向協定
  • 程式化網路管理
  • 靈活的流量調度

SDN 架構層次

  • 基礎設施層：實體/虛擬交換器（OpenFlow vSwitch）
  • 控制層：SDN Controller（如OpenDaylight、ONOS）
  • 應用層：網路應用（防火牆、負载均衡、分析）

SDN 應用：資料中心網路自動化、WAN頻寬調度（SD-WAN）、網路功能虛擬化（NFV）、多雲網路連接。

網際網路核心 — IP定址、子網路切割、NAT、路由協定與IPv6

IP 定址基礎

IP（Internet Protocol）位址是網路中每台設備的邏輯位址，用於識別網路接口：

IPv4 位址：32位元（4 bytes），以點分十進制表示，例如 192.168.1.1。
IPv6 位址：128位元（16 bytes），以冒號十六進制表示，例如 2001:db8::1。

IPv4 位址分類 (Classful Addressing)

Class 前置位元 位址範圍 預設遮罩 用途
A 0 1.0.0.0 - 126.255.255.255 255.0.0.0 (/8) 大型網路
B 10 128.0.0.0 - 191.255.255.255 255.255.0.0 (/16) 中型網路
C 110 192.0.0.0 - 223.255.255.255 255.255.255.0 (/24) 小型網路
D 1110 224.0.0.0 - 239.255.255.255 N/A 多播 (Multicast)
E 1111 240.0.0.0 - 255.255.255.255 N/A 實驗用 (保留)

私有IP位址與保留位址

RFC 1918 定義了私有IP位址範圍，僅用於內部網路：

Class 私有位址範圍 CIDR
A 10.0.0.0 - 10.255.255.255 10.0.0.0/8
B 172.16.0.0 - 172.31.255.255 172.16.0.0/12
C 192.168.0.0 - 192.168.255.255 192.168.0.0/16

特殊保留位址

  • 127.0.0.0/8：Loopback（本機回送），最常用 127.0.0.1
  • 0.0.0.0/8：表示「所有位址」或「未知位址」
  • 169.254.0.0/16：APIPA（自動私有IP位址）
  • 255.255.255.255：有限廣播位址

子網路遮罩與切割

子網路切割（Subnetting）將一個大網路劃分為多個較小的子網路，提高位址利用率和管理效率：

子網路遮罩 (Subnet Mask)：32位元遮罩，網路部分為1，主機部分為0。例如 255.255.255.0 表示前24位元為網路部分。

切割計算

  • 子網路數量 = 2^(借用位元數)
  • 每個子網路可用主機數量 = 2^(剩餘主機位元數) - 2
  • 減2的原因：網路位址（主機部分全0）和廣播位址（主機部分全1）

CIDR 表示法

CIDR（Classless Inter-Domain Routing）使用前綴長度代替分類：

CIDR 子網路遮罩 可用主機數 典型用途
/24 255.255.255.0 254 小型LAN / C類替代
/25 255.255.255.128 126 小型子網路
/26 255.255.255.192 62 部門子網路
/27 255.255.255.224 30 小型部門
/28 255.255.255.240 14 點對點鏈路
/30 255.255.255.252 2 路由器間鏈路

IP 封包結構 (IPv4 Header)

IPv4封包標頭長度最小20 bytes，最大60 bytes：

| Version | IHL | DSCP | ECN | Total Length | | 4 bits | 4B | 6b | 2b | 16 bits | | Identification | Flags | Fragment Offset | | 16 bits | 3b | 13 bits | | TTL | Protocol | Header Checksum | | 8 bits | 8 bits | 16 bits | | Source IP Address (32 bits) | | Destination IP Address (32 bits) | | Options (if any, 0-40 bytes) |
欄位 大小 說明
Version 4 bits IP版本（IPv4 = 4, IPv6 = 6）
IHL 4 bits 標頭長度（最小值5 = 20 bytes）
DSCP 6 bits 差異化服務代碼點（QoS標記）
Total Length 16 bits 封包總長度（含標頭），最大65535 bytes
Identification 16 bits 分片識別碼（同一封包的分片相同）
Flags 3 bits DF（不分片）、MF（更多分片）
Fragment Offset 13 bits 分片在原始封包中的偏移量
TTL 8 bits 存活時間，每經過一台路由器減1，歸零時丟棄
Protocol 8 bits 上層協定（TCP=6, UDP=17, ICMP=1）
Header Checksum 16 bits 標頭錯誤檢查（每跳重新計算）

NAT 網路位址轉譯

NAT（Network Address Translation）將私有IP位址轉換為公有IP位址：

NAT 類型
  • 靜態 NAT：一對一映射（內部IP ⇔ 外部IP）
  • 動態 NAT：私有IP從公有IP池取用
  • PAT (NAPT)：多對一，使用埠號區分（最常見）
NAT 優缺點
  • 節省公有IP位址
  • 隱藏內部網路結構
  • 便于內部網路擴充
  • 破壞端到端連通性
  • 增加處理延遲

PAT (Port Address Translation)：也稱為NAPT（Network Address Port Translation），是家用路由器最常用的NAT方式。將多個內部IP的連線映射到單一外部IP，使用不同的TCP/UDP埠號區分。

路由協定

路由協定讓路由器動態學習網路拓撲，建立路由表：

協定 類型 度量 (Metric) 收斂速度 適用範圍
RIP 距離向量 跳數 (Hop Count) 慢（30秒更新） 小型網路
OSPF 鏈路狀態 成本 (Cost) 快（立即更新） 中型到大型網路
IS-IS 鏈路狀態 成本 (Cost) 快 大型ISP網路
EIGRP 混合型 複合度量 快 思科網路
BGP 路徑向量 AS路徑長度 慢（增量更新） 網際網路骨幹

IGP vs EGP：IGP（內部閘道協定，如OSPF/RIP）在同一個AS內運作；EGP（外部閘道協定，如BGP）在不同AS之間運作。AS（Autonomous System）是由單一組織管理的網路集合。

IPv6 簡介

IPv6 解決 IPv4 位址枯竭問題，提供更多功能和改進：

特性 IPv4 IPv6
位址長度 32 bits 128 bits
位址數量 ~43億 ~3.4×10^38
表示法 點分十進制 冒號十六進制
自動設定 DHCP SLAAC（無狀態自動設定）
安全性 可選（IPSec） 內建（IPSec必備）
分片 路由器可分片 僅傳送端可分片
廣播 有廣播位址 無廣播，使用多播取代
標頭 可變長度（20-60 bytes） 固定長度（40 bytes）

IPv6 位址簡寫規則：
1. 每組前導零可省略：2001:0db8::1 → 2001:db8::1
2. 連續一組或多組0可用 :: 取代（僅限一次）：2001:0db8:0000:0000:0000:0000:0000:0001 → 2001:db8::1

DNS 域名系統

DNS（Domain Name System）將人類可讀的域名轉換為IP位址：

  • 階層式結構：根網域 → 頂級網域(.com/.org/.tw) → 二級網域 → 子網域
  • 遞迴查詢：DNS伺服器代表客戶端完成完整查詢
  • 迭代查詢：DNS伺服器引導客戶端到下一級伺服器
  • DNS紀錄類型：
    • A：IPv4位址
    • AAAA：IPv6位址
    • CNAME：別名（規範名稱）
    • MX：郵件交換伺服器
    • NS：名稱伺服器
    • PTR：反向查詢（IP → 域名）

DNS 快取：DNS查詢結果會在各級伺服器和客戶端快取，TTL（Time to Live）欄位控制快取時間，減少網路負載。
